# 关键参数说明

本文档解释当前配置文件中的 3 个关键参数：

- `nomoto_T_s`
- `nomoto_K`
- `speed_tau_s`

它们定义于 [default_config.json](/home/sun/projects/cpp/ship-motion-sim/config/default_config.json#L2)，实际生效位置在 [simple_nomoto_ship_model.cpp](/home/sun/projects/cpp/ship-motion-sim/src/model/simple_nomoto_ship_model.cpp#L88)，对应的模型设计说明见 [final.md](/home/sun/projects/cpp/ship-motion-sim/final.md#L70)。

需要先明确一点：

- 这 3 个值现在是“原型行为参数”
- 不是某型实船经过辨识得到的真实标定参数

它们的目标是让当前最小模型在数值上稳定、行为上合理、观感上容易验证。

## 1. 参数总览

当前默认值为：

```json
"nomoto_T_s": 10.0,
"nomoto_K": 0.08,
"speed_tau_s": 8.0
```

它们分别控制：

- `nomoto_T_s`：转向响应建立得有多慢
- `nomoto_K`：固定舵角最终能建立多大的转向速度
- `speed_tau_s`：车令变化后速度建立得有多慢

可以把它们理解为两个一阶系统的“快慢”和“强弱”参数。

## 2. `nomoto_T_s`

## 2.1 数学位置

它出现在一阶 Nomoto 航向模型中：

```text
r_dot = (K * delta - r) / T
psi_dot = r
```

对应代码：

- [simple_nomoto_ship_model.cpp](/home/sun/projects/cpp/ship-motion-sim/src/model/simple_nomoto_ship_model.cpp#L93)

其中：

- `delta`：舵角
- `r`：转首角速度
- `T`：也就是 `nomoto_T_s`

## 2.2 物理意义

`nomoto_T_s` 是转向惯性的时间常数。

它不表示“10 秒转完一次弯”，而表示：

- 船在固定舵角作用下，转首角速度 `r` 接近目标值的快慢

对一阶系统来说：

- 经过 `1*T`，大约达到最终值的 63%
- 经过 `3*T`，大约达到最终值的 95%

所以当 `T = 10s` 时，表示船的转向建立是“十秒量级”的，而不是一打舵就瞬间出现稳定转向。

## 2.3 对行为的影响

- `T` 越小：转向建立越快，船显得更灵敏
- `T` 越大：转向建立越慢，船显得更钝重

它主要控制的是：

- “多久转起来”

而不是：

- “最终能转多快”

## 2.4 为什么当前取 `10.0`

当前取 `10.0` 的原因是平衡了 3 件事：

1. 需要明显看到转向惯性
   如果 `T` 太小，比如 `1` 或 `2`，船会显得像轻小艇，打舵后几乎立刻建立转向。

2. 需要在演示场景里看得出变化
   如果 `T` 太大，比如 `30`，在 20 秒到 3 分钟的示例场景里转向建立会太慢，行为不明显。

3. 需要和当前积分步长稳定配合
   当前 `dt = 0.1s`，所以 `dt / T = 0.01`，对显式欧拉积分来说比较温和。

因此 `10s` 是一个偏“中间值”的演示参数：

- 有明显惯性
- 不会太快
- 也不会慢得看不出来

## 3. `nomoto_K`

## 3.1 数学位置

它也出现在 Nomoto 模型中：

```text
r_dot = (K * delta - r) / T
```

对应代码：

- [simple_nomoto_ship_model.cpp](/home/sun/projects/cpp/ship-motion-sim/src/model/simple_nomoto_ship_model.cpp#L93)

## 3.2 物理意义

`nomoto_K` 是舵效增益。

在稳态下，`r_dot = 0`，所以：

```text
r_ss = K * delta
```

这表示：

- 在给定固定舵角时，最终会建立多大的转首角速度

因此 `K` 决定的是：

- “同样的舵角最终能转得多快”

## 3.3 用当前默认值举例

如果舵角是 `10 deg`，换算成弧度约为 `0.1745 rad`。

当前 `K = 0.08` 时：

```text
r_ss = 0.08 * 0.1745 = 0.01396 rad/s
```

换算成角度大约为：

```text
0.01396 * 180 / pi ≈ 0.8 deg/s
```

这表示固定 `10 deg` 舵角下，最终会趋向每秒约 `0.8 deg` 的转向速度。

如果是 `30 deg` 舵角，则最终稳态转向速度大约为 `2.4 deg/s`。

这个量级对当前原型来说比较合适：

- 不会几乎不转
- 也不会转得过猛

## 3.4 对行为的影响

- `K` 越小：同样舵角下最终转得更慢，轨迹更缓
- `K` 越大：同样舵角下最终转得更快，轨迹更急

它主要控制的是：

- “最终能转多快”

而不是：

- “多久建立起来”

这点要和 `T` 区分开。

## 3.5 为什么当前取 `0.08`

当前取 `0.08` 是为了让“固定舵角 -> 可观察的持续转向”这个关系落在一个比较合理的范围：

- 如果太小，比如 `0.02`
  打舵后轨迹很平，转向效果不明显

- 如果太大，比如 `0.2`
  在当前简化模型下会显得过于灵敏，轨迹弯曲过猛

`0.08` 配合 `T = 10s` 后，能形成一个比较平衡的组合：

- 舵角有足够作用
- 转向又不是瞬时完成
- 在 30 秒和 3 分钟场景里都容易看清行为

## 4. `speed_tau_s`

## 4.1 数学位置

它出现在速度模型中：

```text
u_dot = (u_cmd - u) / tau_u
```

对应代码：

- [simple_nomoto_ship_model.cpp](/home/sun/projects/cpp/ship-motion-sim/src/model/simple_nomoto_ship_model.cpp#L88)

其中：

- `u_cmd`：车令映射出的目标速度
- `u`：当前速度
- `tau_u`：也就是 `speed_tau_s`

## 4.2 物理意义

`speed_tau_s` 是速度响应时间常数。

它描述的不是“几秒钟一定达到目标速度”，而是：

- 当前速度接近目标速度的快慢

同样地，对一阶系统来说：

- 经过 `1*tau_u`，大约达到目标变化量的 63%
- 经过 `3*tau_u`，大约达到目标变化量的 95%

如果从 `0` 加速到 `3.0 m/s`，且 `tau_u = 8s`，那么：

- `8s` 后大约到 `1.9 m/s`
- `24s` 后大约到 `2.85 m/s`

## 4.3 对行为的影响

- `tau_u` 越小：加减速更快
- `tau_u` 越大：加减速更慢

它控制的是：

- “车令生效后速度建立有多快”

## 4.4 为什么当前取 `8.0`

当前取 `8.0` 的理由是：

1. 不能让速度瞬间跳变
   如果太小，比如 `1` 或 `2`，一给车令速度就很快跳起来，不像船的惯性。

2. 也不能太迟缓
   如果太大，比如 `20`，短时间场景里很难看出车令是否有效。

3. 希望速度建立略快于转向建立
   当前 `speed_tau_s = 8`，`nomoto_T_s = 10`，意味着船通常会先逐步动起来，再逐步建立更充分的转向响应。

这个组合在观感上更自然：

- 船先开始加速
- 然后再逐步出现明显的转向曲线

## 5. 这 3 个值为什么要配合看

单独解释每个参数不够，因为它们实际是组合生效的。

当前组合是：

```json
"nomoto_T_s": 10.0,
"nomoto_K": 0.08,
"speed_tau_s": 8.0
```

这组组合的设计目标是：

1. 速度不会瞬间跳变
2. 航向不会瞬间跳变
3. 固定舵角下能看到明显转向
4. 回舵后能看到转首角速度一阶衰减
5. 在 30 秒到 3 分钟的演示场景里，轨迹变化足够明显
6. 对当前 `dt = 0.1s` 的欧拉积分足够稳定

从数值角度看：

- `dt / T = 0.1 / 10 = 0.01`
- `dt / tau_u = 0.1 / 8 = 0.0125`

这两个比例都比较小，所以对当前显式欧拉更新来说是温和的。

## 6. 为什么它们不是“真实船舶物理常数”

当前模型有几个明确的简化：

- `K`、`T` 不随航速变化
- 没有舵机动态
- 没有横荡自由度
- 没有推进器、阻力、风浪流
- 没有转向与航速耦合

因此这 3 个值当前只能被理解为：

- 原型行为参数

不能直接理解为：

- 某型实船辨识得到的工程参数

这一点在设计文档中已经说明，见 [final.md](/home/sun/projects/cpp/ship-motion-sim/final.md#L84) 和 [final.md](/home/sun/projects/cpp/ship-motion-sim/final.md#L105)。

## 7. 调参时可以怎么理解

如果想调成不同风格的船，可以这样理解：

### 更像大船

- `T` 更大
- `K` 稍小
- `tau_u` 更大

效果：

- 转向慢
- 加速慢
- 航迹更平缓、更迟钝

### 更像小艇

- `T` 更小
- `K` 更大
- `tau_u` 更小

效果：

- 响应快
- 转向急
- 加速快

### 只想让弯更大

优先调 `K`。

### 只想让转向建立更快或更慢

优先调 `T`。

### 只想改变加减速快慢

优先调 `speed_tau_s`。

## 8. 如何验证这 3 个参数是否合适

最可靠的方式不是只看数字，而是看 3 类场景输出：

### 场景 1：只给车令，不打舵

看速度曲线是否符合预期：

- 是否逐步上升
- 是否没有明显跳变

### 场景 2：固定车令，固定舵角

看航向曲线和角速度曲线：

- 角速度是否逐渐建立
- 航向是否持续变化

### 场景 3：打一段舵后回正

看回舵后角速度是否逐渐衰减：

- 是否不会瞬间归零
- 是否符合一阶惯性直觉

## 9. 总结

这 3 个参数的核心作用可以简化为：

- `nomoto_T_s`：控制转向响应建立得多快
- `nomoto_K`：控制固定舵角最终转得多快
- `speed_tau_s`：控制速度响应建立得多快

当前默认值：

```json
"nomoto_T_s": 10.0,
"nomoto_K": 0.08,
"speed_tau_s": 8.0
```

选择它们的原因不是“真实船型标定”，而是为了让当前最小原型具备：

- 稳定的数值行为
- 合理的运动观感
- 容易观察和验证的轨迹变化

如果未来项目引入舵机动态、航速相关参数或 3-DOF 模型，这 3 个值的解释方式和推荐范围都需要随之升级。
